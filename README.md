## 🚀 Featured Work

### 📦 VTSC PaintPro - Powder Coating Warehouse & Business Management System
*Role: System Analyst & Back-End Developer | 03/2026 – 06/2026*

A product distribution management system designed to track performance and control real-time inventory utilizing a distributed architecture.

- **Tech Stack:** Next.js (React), Node.js, Express.js, MongoDB Atlas, JWT, Tailwind CSS.
- **Key Responsibilities & Achievements:**
  - **System & Database Design:** Analyzed and designed the entire NoSQL database architecture. Applied embedded document techniques to optimize queries for products and color variants (SKUs), significantly reducing latency compared to traditional SQL.
  - **Authentication & Security:** Built a highly secure authentication flow using `Bcrypt` for password hashing and **Dual JWT** (15-minute Access Token & 7-day Refresh Token). Integrated `Axios Interceptors` for silent token renewal, ensuring a seamless user experience.
  - **Inventory Management:** Developed business logic for inventory management down to the SKU level (available vs. on-hold stock) with automated calculation and data reconciliation during transactions.
  - **Traceability:** Integrated dynamic QR code generation and stored documentation on the decentralized IPFS network, enabling customers to trace product origins and expiration dates.

**🔥 Key Features & Technical Highlights:**
- **Smart Contract Escrow:** Developed immutable Solidity contracts on the Sepolia Testnet to lock funds and auto-execute payments/penalties, eliminating third-party reliance.
- **Immutable Proof of Delivery:** Integrated **IPFS (Pinata)** to store delivery evidence (documents, images), ensuring documents are tamper-proof via unique Content Identifiers (CIDs).
- **Hybrid Web2.5 Architecture (Gas Optimized):** Strategically decoupled on-chain logic (finances, rules) from off-chain data (GPS tracking history, email notifications) using Node.js & MongoDB to achieve a seamless UX with minimal gas fees.
- **Concurrency & Security:** Implemented strict state management on-chain to prevent race conditions (e.g., multiple drivers accepting the same contract simultaneously) and secured user identity via MetaMask.
